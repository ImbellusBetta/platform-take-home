Help.

This is a project I made for Imbellus. It is a basic web app with the following functionality: geoencode a location, reverse geoencode a location, and calculate the geometric distance between two points. What follows are some basic instructions on using the various features of the application.

Geoencoding.

Geocoding is the process of converting addresses (like a street address) into geographic coordinates (like latitude and longitude). This RESTful tool makes calls to Google's Geocoding API. By entering an address (whether it be a street address, or a location), the API can obtain a latitude longitude pair. For more information, see here.

Reverse Geoencoding.

Reverse geocoding is the exact opposite process from above. Simply enter a value in degrees for latitude and longitude to see what lies at that location on Earth! The only rules for these values is that latitude must be between -90 and 90 degress, and longitude must be between -180 and 180 degrees. For more information, see here.

Geometric Distance.

Calculating the geometric distance between two locations makes use of the Haversine formula. This is used to calculate determines the great-circle distance between two points on a sphere given their longitudes and latitudes.
